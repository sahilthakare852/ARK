## Format note

Markdown was chosen over slides deliberately. This team works in Bitbucket Cloud and thinks in Git workflows. Documentation that lives where engineers read it — in a repo, rendered natively in the browser — is a more honest representation of how I'd actually communicate on this team than a deck produced for the exercise. The one exception is Deliverable 7 (leadership summary), which is also provided as a PDF since a CPEO brief is the one artifact that benefits from a polished single-page format.

## Reading order

Start with Deliverable 1. Every subsequent deliverable builds on it directly. The system design decisions inform the build strategy (D2), the epics (D3), the phasing (D4), and the team assignments (D5). Reading them in order makes the reasoning traceable.

The leadership summary (D7) is designed to be read independently in 90 seconds, without the context of the other deliverables.
